Reports a named-argument key that does not conform to KotlinPoet's required naming pattern. Keys must match the regular expression [a-z]+[\w_]* — they must start with a lowercase ASCII letter.

Passing an invalid key to addNamed causes an IllegalArgumentException at runtime (mirroring the check inside CodeBlock.addNamed).

Example (flagged):


CodeBlock.builder().addNamed(
    "%Food:L",
    mapOf("Food" to "tacos")  // 'Food' starts with uppercase → runtime exception
)

Correct usage:


CodeBlock.builder().addNamed(
    "%food:L",
    mapOf("food" to "tacos")
)

Quick fix: Rename 'Food' to 'food' – converts the key to lowercase and updates the matching placeholder in the format string simultaneously.
