Reports non-blocking syntax issues found in a KotlinPoet format string during parsing. Unlike mixed-style errors (which prevent analysis entirely), these issues allow the rest of the format string and its arguments to be validated normally.

IssueExampleQuick fix
Dangling % "total: %" Escape as %% · Remove %
Unknown placeholder type "%Z value" Remove %Z
Invalid positional index (must be ≥ 1) "%0L" Fix index to 1 → "%1L"

Dangling % — example:


CodeBlock.builder().add("100% done")
//                           ^ dangling: use %% for a literal percent sign

Corrected:


CodeBlock.builder().add("100%% done")

Unknown placeholder type — example:


CodeBlock.builder().add("%Z value", someArg)
//                       ^^ not a valid KotlinPoet placeholder

Invalid positional index — example:


CodeBlock.builder().add("%0L", someArg)
//                       ^^^ index must be 1-based (≥ 1)
