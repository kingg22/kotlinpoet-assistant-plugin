Tu plugin ya implementa correctamente:

* `Annotator` → errores y highlighting básico

Esto ya cubre **la base semántica**.

# 1️⃣ Highlight mejorado de placeholders y argumentos

## API

`Annotator` (ya lo usas)

```kotlin
com.intellij.lang.annotation.Annotator
```

## Qué hace

Permite agregar:

* highlight de texto
* warnings/errors
* attributes de color
* quick fixes

## Mejora recomendada

Actualmente usas:

```kotlin
DefaultLanguageHighlighterColors.HIGHLIGHTED_REFERENCE
```

Esto funciona, pero es **muy genérico**.

### Mejor opción

Crear un `TextAttributesKey` propio.

```kotlin
object KotlinPoetHighlightKeys {

    val PLACEHOLDER = TextAttributesKey.createTextAttributesKey(
        "KPOET_PLACEHOLDER",
        DefaultLanguageHighlighterColors.KEYWORD
    )

    val ARGUMENT = TextAttributesKey.createTextAttributesKey(
        "KPOET_ARGUMENT",
        DefaultLanguageHighlighterColors.HIGHLIGHTED_REFERENCE
    )
}
```

Luego usarlo:

```kotlin
holder.newSilentAnnotation(HighlightSeverity.INFORMATION)
    .range(range)
    .textAttributes(KotlinPoetHighlightKeys.PLACEHOLDER)
    .create()
```

## Resultado visual

Antes:

```
"%N = %N()", a, b
```

Después:

```
"%N = %N()", a, b
 ↑    ↑
highlight azul
```

# 2️⃣ Highlight del argumento vinculado

Actualmente el highlight ocurre **solo cuando el usuario usa usages**.

Puedes hacerlo permanente.

## API

`Annotator`

## Idea

Cuando haces el binding:

```kotlin
boundContext.forEach { (placeholder, arg) -> }
```

highlight también el argumento:

```kotlin
arg.expression.textRange
```

Ejemplo:

```kotlin
holder.newSilentAnnotation(HighlightSeverity.INFORMATION)
    .range(arg.expression.textRange)
    .textAttributes(KotlinPoetHighlightKeys.ARGUMENT)
    .create()
```

## Resultado visual

```
addStatement("%N = %N()", localName, typeName)
               │      │
               │      └ highlight cyan
               └ highlight cyan
```

Similar a `printf` de C.

# 3️⃣ Inlay hints para significado del placeholder

## API

```text
com.intellij.codeInsight.hints.InlayHintsProvider
```

## Qué hace

Inserta texto visual **inline en el editor**.

Ejemplo real del IDE:

```
println(x)  // parameter hint
        │
        value:
```

## En tu plugin

Podrías mostrar:

```
"%N = %N()", localName, typeName
  name     constructor
```

### Ejemplo

```kotlin
class KotlinPoetInlayHintsProvider : InlayHintsProvider<NoSettings>
```

Collector:

```kotlin
sink.addInlineElement(
    offset,
    false,
    factory.smallText("name"),
    false
)
```

## UX recomendada

Solo mostrar hints cuando:

* cursor esté dentro del call
* o línea visible

# 4️⃣ Preview del código generado (feature estrella)

Esto es **muy poderoso para KotlinPoet**.

## API

```text
InlayHintsProvider
```

pero usando:

```text
block / collapsible presentation
```

## Qué hace

Permite mostrar:

```
▶ Generated Kotlin
```

expandible a:

```kotlin
fun hello() {
    println("Hi")
}
```

## Implementación

Generas preview:

```kotlin
// ejemplo
val preview = codeBlock.toString()
```

presentación:

```kotlin
factory.collapsibleList(
    listOf(factory.smallText("Generated Kotlin")),
    factory.smallText(preview)
)
```

## Resultado visual

```
addStatement("%N = %N()", a, b)

▶ Generated Kotlin
```

# 6️⃣ Mejoras a tu Annotator actual

Tu annotator es muy bueno, pero puedes mejorar:

## Problema

Actualmente repites:

```
span → ranges → annotations
```

Puedes extraer helper:

```kotlin
fun AnnotationHolder.highlight(range: TextRange, key: TextAttributesKey)
```

---

## Mejora de rendimiento

Mover:

```
extractorRegistry.extract
bindingResolver.bind
```

a:

```
CachedValue
```

porque el annotator corre **muchas veces por segundo**.
